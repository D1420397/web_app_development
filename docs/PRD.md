# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述

### 背景與動機
現代人生活忙碌，常常在買了食材後不知道該煮什麼，或者是看到喜歡的食譜卻難以系統化地整理與收藏。本系統旨在提供一個集中管理的平台，讓使用者能根據現有的食材快速找到合適的食譜，並建立個人的料理資料庫。

### 目標用戶
*   **一般使用者**：忙碌的上班族、家庭主婦/主夫、料理愛好者。
*   **管理員**：負責維護系統食譜品質、管理分類標籤與監控系統運行的維運人員。

### 核心價值主張
讓「想吃什麼」與「現有食材」之間不再有距離，透過直覺的食材搜尋與收藏功能，提升下廚的便利性與樂趣。

---

## 2. 功能需求

以下列出系統的主要功能需求與使用者故事：

| 編號 | 功能名稱 | 使用者故事 (User Story) |
| :--- | :--- | :--- |
| FR-01 | 食材收藏/儲存 | 作為 **一般使用者**，我希望 **能夠儲存我家中的常備食材或是剛買回來的食材**，以便 **記錄目前現有的資源**。 |
| FR-02 | 食材組合搜尋 | 作為 **一般使用者**，我希望 **能勾選多種食材組合進行搜尋**，以便 **找到能消耗現有食材的對應食譜**。 |
| FR-03 | 食譜發佈與管理 | 作為 **一般使用者**，我希望 **能上傳自己的私房食譜（包含標題、步驟、照片、所需食材）**，以便 **建立個人化的線上菜單**。 |
| FR-04 | 個人收藏清單 | 作為 **一般使用者**，我希望 **能將別人分享的優質食譜加入收藏夾**，以便 **日後快速查看與參考**。 |
| FR-05 | 分類與標籤系統 | 作為 **管理員**，我希望 **能定義食譜的分類（如：早餐、甜點、低卡）**，以便 **使用者能更精準地分類搜尋**。 |
| FR-06 | 食譜審核與維護 | 作為 **管理員**，我希望 **能編輯或刪除內容不當、重複或錯誤的食譜**，以便 **維護社群內容的品質**。 |

## 3. 非功能需求

### 技術限制
*   **後端框架**：使用 Python Flask。
*   **模板引擎**：使用 Jinja2 渲染動態頁面。
*   **資料庫**：使用 SQLite，便於輕量開發與部署。
*   **前端**：使用 Vanilla CSS 與 JavaScript (可根據需要加入現代化 UI 元件)。

### 效能與安全考量
*   **回應速度**：搜尋食材組合時，系統應於 1 秒內返回結果。
*   **資料保存**：確保使用者建立的食譜與收藏資料在資料庫中持久保存，並有基本防護防止 SQL Injection。
*   **權限控制**：管理員頁面需有身份驗證機制，防止一般使用者存取。

## 4. MVP 範圍

為了讓專案能快速上線測試，我們將功能劃分為三個層級：

### Must Have (必要功能)
*   使用者註冊與登入。
*   食材資料表維護與新增食材。
*   食譜新增、編輯、刪除（CRUD）。
*   基礎食材組合搜尋（包含特定食材的食譜）。

### Should Have (重要功能)
*   食譜圖片上傳。
*   個人收藏（Like）功能。
*   食譜分類標籤（如：中式、西式）。

### Nice to Have (進階功能)
*   食譜留言與評分系統。
*   食材剩餘量提醒（即將過期提醒）。
*   AI 輔助食材識別（上傳食材照片自動識別種類）。
